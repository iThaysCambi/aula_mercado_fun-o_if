# 🛒 Validação de Fluxo de Caixa e Condicionais de Mercado (Estruturas `if/else`)

Este módulo contém a implementação de algoritmos de tomada de decisão lógica voltados para cenários de negócios e controle de fluxo de caixa mercantil. A aplicação simula operações de PDV (Ponto de Venda) e tomadas de decisão automatizadas com base em regras de faturamento e validação de saldos.

Desenvolvido como atividade prática de consolidação de fundamentos durante o Bootcamp de Análise de Dados na **Generation Brasil**.

---

## 🎯 Contexto e Objetivos Técnicos

O objetivo principal desta atividade foi aplicar conceitos de estruturas condicionais encadeadas para resolver problemas de lógica operacional de mercado:

*   **Tomada de Decisão Automatizada:** Implementação de desvios condicionais com estruturas `if`, `elif` e `else` para avaliar cenários de compras e balanços.
*   **Tratamento de Dados de Entrada:** Captura e conversão de inputs de usuário para tipos numéricos adequados (`float` e `int`), garantindo a precisão dos cálculos financeiros de mercado.
*   **Simulação de Negócio:** Modelagem de algoritmos que validam se um estabelecimento possui saldo suficiente para compras de suprimentos ou se um cliente atingiu os critérios para liberação de benefícios/descontos.

---

## 🛠️ Tecnologias e Ferramentas Utilizadas

*   **Linguagem:** Python 3.x
*   **Ambiente de Desenvolvimento:** Jupyter Notebook (`.ipynb`)
*   **Paradigmas:** Programação Estruturada e Lógica Condicional

---

## 📂 Como Executar o Script

1.  Certifique-se de ter o Python e o ambiente do Jupyter (ou VS Code com extensão Jupyter) instalados.
2.  Abra o arquivo `aula_mercado_fun-o_if.ipynb`.
3.  Execute as células sequencialmente para interagir com o terminal simulado de operações de mercado.

---

## 🚀 Diferenciais de Negócio Demonstrados

Embora seja um projeto de fundamentos, a arquitetura lógica simula a base de motores de regras de crédito, sistemas de checkout de e-commerce e ferramentas elementares de ERP (Enterprise Resource Planning), provando a capacidade de traduzir regras de negócio complexas em código limpo, documentado e de fácil manutenção.
